## 2. 总主线

当前服务器项目的价值不只是写出一个 Echo Server，而是通过不断演进理解服务器为什么这样设计。

```text
socket API
    -> 阻塞 I/O
    -> 多线程/线程池
    -> I/O 多路复用
    -> fd 生命周期管理
    -> 业务与 I/O 分离
    -> HTTP 协议解析
    -> Reactor 模型
    -> 后端工程化
```

每一层都解决上一层的问题，同时引入新的问题。

| 阶段 | 解决的问题 | 引入的问题 |
|---|---|---|
| 阻塞 Echo Server | 理解 socket 基础流程 | 一次只能处理一个客户端 |
| 每连接一线程 | 多客户端并发 | 线程数量不可控 |
| 线程池阻塞模型 | 控制线程数量 | 慢连接占用 worker |
| select | 单线程监听多个 fd | 需要维护 fd_set、max_fd、client_fds |
| select + threadpool | I/O 与业务初步分离 | fd 生命周期、并发 write、任务边界风险 |
| poll / epoll | 改善事件通知机制 | 连接状态管理仍然复杂 |
| HTTP Server | 从 socket demo 进入后端服务 | 协议解析、缓冲区、状态、错误码、日志 |

## 3. 当前项目中的两个核心版本

### 3.1 threadpool 版本

目录：

```text
project/server/threadpool
```

模型：

```text
main 线程：
    accept_client()
    pool.submit([client_fd] { handle(client_fd); })

worker 线程：
    while (read(client_fd))
    write_all(client_fd)
    close(client_fd)
```

特点：

```text
一个连接被提交给一个 worker。
worker 长期负责这个连接的 read/write/close。
```

适合学习：

```text
线程池
任务队列
阻塞 read
client_fd 所有权转移
每连接一个任务
```

局限：

```text
如果客户端连接后长期不发数据，worker 会阻塞在 read。
线程池大小固定，慢连接多了以后 worker 会被耗尽。
```

### 3.2 select_pool 版本

目录：

```text
project/server/select_pool
```

模型：

```text
main：
    创建 TcpServer
    创建 ThreadPool
    创建 Selector
    selector.loop()

Selector：
    select()
    accept_client()
    recv 一次
    提交 data 给 ThreadPool
    管理 client_fd 集合和关闭

ThreadPool：
    处理 data
    当前简化版中直接 write 回 client_fd
```

特点：

```text
Selector 负责 I/O 就绪事件。
ThreadPool 负责业务任务。
client_fd 生命周期主要由 Selector 管理。
```

这是当前更接近后续 Reactor 的版本，但仍是学习阶段简化模型。

## 4. 当前阶段核心口诀

```text
I/O 状态归 Selector
业务耗时归 Pool
资源生命周期归拥有者
main 只负责组装
```

更具体地说：

```text
TcpServer：负责 listen_fd 的创建、绑定、监听、关闭。
Selector：负责 client_fd 集合、select、recv、remove_client。
ThreadPool：负责 worker 线程和任务队列。
BusinessHandler：负责 data -> response。
```

## 5. 当前必须掌握

```text
socket / bind / listen / accept
read / write / recv / send
fd 是进程 fd 表中的索引
TCP 是字节流，没有消息边界
select 会修改 fd_set
每轮 select 前必须 read_set = master_set
recv 返回 0 表示对端有序关闭
close fd 后必须同步更新 Selector 的内部集合
线程池执行 task 必须在锁外
std::string(buffer, bytes_read) 不依赖 \0
write_all 需要处理短写和 EINTR
```
