# 10 测试、调试与压测

## 1. nc 连接测试

基本测试：

```bash
nc 127.0.0.1 8080
```

如果需要让 Ctrl+D 关闭网络写方向：

```bash
nc -N 127.0.0.1 8080
```

注意：普通 `nc` 的 Ctrl+D 不一定立刻关闭 TCP socket。服务端是否能收到 FIN，取决于 nc 实现和参数。

## 2. 多客户端测试

```text
1. 同时打开多个 nc。
2. 每个客户端发送不同内容。
3. 验证 echo 是否互不干扰。
4. 关闭其中一个客户端，观察服务端是否正确 remove_client。
```

## 3. 断开测试

重点观察：

```text
recv 返回 0。
closed_fds_ 是否记录该 fd。
remove_client 是否 close + FD_CLR + erase。
closed_fds_ 是否 clear。
max_fd_ 是否正确更新。
```

## 4. 线程数观察

```bash
ps -T -p PID
ls /proc/$(pidof server)/task | wc -l
```

适合对比：

```text
每连接一线程模型
线程池模型
select + threadpool 模型
```

## 5. 上下文切换观察

```bash
pidstat -w -p PID 1
```

对比指标：

```text
线程数量
内存占用
CPU 使用率
上下文切换
QPS
平均延迟
P95 / P99 延迟
失败数
```

## 6. GDB 调试建议

常用命令：

```text
b 函数名 / 行号
r
n
s
p 变量名
bt
```

服务器调试重点：

```text
client_fds_ 当前有哪些 fd？
max_fd_ 是否正确？
closed_fds_ 是否重复保留旧 fd？
select 返回值是多少？
errno 是什么？
```
