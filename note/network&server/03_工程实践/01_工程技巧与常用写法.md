# 09 工程技巧与常用写法

## 1. 用实际读取长度构造 std::string

场景：

```cpp
char buffer[1024];
ssize_t n = recv(fd, buffer, sizeof(buffer) - 1, 0);
```

推荐：

```cpp
std::string data(buffer, n);
```

原因：

```text
不依赖 '\0'。
只使用本次实际读到的字节。
适合 read/recv 读出的原始字节流。
```

不推荐：

```cpp
std::string data(buffer);
std::string data(buffer, sizeof(buffer) - 1);
```

前者依赖 C 字符串结束符，后者可能把未读到的区域也放进字符串。

## 2. select 每轮复制 fd_set

```cpp
read_set_ = master_set_;
select(max_fd_ + 1, &read_set_, nullptr, nullptr, nullptr);
```

原因：

```text
select 会修改传入的 fd_set。
master_set 保存长期监听集合。
read_set 是本轮临时结果集合。
```

## 3. 任务队列锁外执行 task

```cpp
std::function<void()> task;
{
    std::unique_lock<std::mutex> lock(mutex_);
    condition_.wait(lock, [this] {
        return stop_ || !tasks_.empty();
    });
    task = std::move(tasks_.front());
    tasks_.pop();
}

task();
```

原因：

```text
锁只保护任务队列。
task 可能阻塞，不能在持锁状态下执行。
```

## 4. write_all 处理短写和 EINTR

工程中不要假设一次 write/send 写完所有数据。

简化思路：

```text
维护已写偏移。
write 返回正数就前进。
errno == EINTR 则重试。
其他错误则失败。
```

## 5. 错误处理统一风格

系统调用常见风格：

```text
成功：返回非负值或有效 fd。
失败：返回 -1，并设置 errno。
```

代码中应统一处理：

```cpp
if (ret == -1) {
    std::cerr << "xxx failed: " << std::strerror(errno) << std::endl;
}
```

## 6. 注释写职责而不是流水账

推荐：

```cpp
// Selector 线程调用：处理一次 client_fd 可读事件。
// 负责 recv 和关闭判断；不做耗时业务。
void Selector::handle_client_event(int client_fd);
```

不推荐：

```cpp
// 处理客户端
```
