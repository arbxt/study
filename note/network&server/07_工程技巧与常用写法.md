# 工程技巧与常用写法

## 1. 用实际读取长度构造 std::string

场景：

```cpp
char buffer[1024];
ssize_t bytes_read = recv(client_fd, buffer, sizeof(buffer) - 1, 0);
```

推荐：

```cpp
std::string data(buffer, bytes_read);
```

原因：

```text
按实际读取到的字节数构造。
不依赖 \0。
适合 socket/read/recv 读到的原始字节。
```

不推荐：

```cpp
std::string data(buffer);
```

问题：

```text
依赖 C 字符串以 \0 结束。
如果数据中包含 \0，或者没有正确终止，会出错。
```

也不推荐：

```cpp
std::string data(buffer, sizeof(buffer) - 1);
```

问题：

```text
会把未实际读取的 buffer 空间也放入字符串。
正确长度应该来自 read/recv 返回值。
```

## 2. write_all 处理短写

`write` 不保证一次写完所有数据。

推荐：

```cpp
static bool write_all(int fd, const char* buffer, ssize_t length) {
    ssize_t total_written = 0;

    while (total_written < length) {
        ssize_t written = write(fd, buffer + total_written, length - total_written);

        if (written > 0) {
            total_written += written;
        } else if (written == -1) {
            if (errno == EINTR) {
                continue;
            }
            return false;
        }
    }

    return true;
}
```

意义：

```text
处理短写。
处理 EINTR。
避免只写出部分 response。
```

后续非阻塞 socket 中还要处理：

```text
EAGAIN / EWOULDBLOCK
写缓冲区
EPOLLOUT
```

## 3. select 每轮复制 fd_set

```cpp
read_set_ = master_set_;
select(max_fd_ + 1, &read_set_, nullptr, nullptr, nullptr);
```

原因：

```text
select 会修改 read_set_。
master_set_ 必须保留完整监听集合。
```

## 4. 删除 fd 时四件套

```cpp
close(fd);
FD_CLR(fd, &master_set_);
erase fd from client_fds_;
if (fd == max_fd_) rebuild_max_fd();
```

口诀：

```text
关闭内核资源，也要同步应用层状态。
```

## 5. lambda 捕获 fd 用值捕获

推荐：

```cpp
pool.submit([client_fd] {
    handle(client_fd);
});
```

不要写：

```cpp
pool.submit([&] {
    handle(client_fd);
});
```

原因：

```text
client_fd 是循环局部变量。
引用捕获可能导致任务拿到被后续循环修改后的值。
```

## 6. 日志输出加锁

教学阶段：

```cpp
std::mutex cout_mutex;
std::lock_guard<std::mutex> lock(cout_mutex);
std::cout << message << std::endl;
```

作用：

```text
避免多个线程输出交叉。
```

工程阶段：

```text
日志队列 + 单独日志线程
日志级别
日志文件滚动
时间戳
线程 id
连接 id
```

## 7. RAII 封装 fd

当前 `TcpServer` 析构中关闭 `listen_fd_`：

```cpp
TcpServer::~TcpServer() {
    if (listen_fd_ != -1) {
        close(listen_fd_);
        listen_fd_ = -1;
    }
}
```

这是 RAII 思想：

```text
对象生命周期结束时释放资源。
```

后续可以进一步封装：

```cpp
class UniqueFd {
public:
    explicit UniqueFd(int fd = -1) : fd_(fd) {}
    ~UniqueFd() { if (fd_ != -1) close(fd_); }
    UniqueFd(const UniqueFd&) = delete;
    UniqueFd& operator=(const UniqueFd&) = delete;
    UniqueFd(UniqueFd&& other) noexcept;
    UniqueFd& operator=(UniqueFd&& other) noexcept;
private:
    int fd_;
};
```

这是把 Linux fd 管理变成 C++ 风格的重要方向。
