# network&server 笔记索引

> 本目录用于沉淀 `study` 项目中的网络编程与服务器学习记录。  
> 当前整理原则：以“服务器模型演进”为主线，以“核心机制”和“工程实践”为复习支撑。  
> 代码本地仓库单独维护，本目录不保留项目代码索引。

## 1. 推荐阅读顺序

### 第一部分：服务器模型演进

1. `01_服务器模型演进/01_阻塞_多进程_多线程_线程池服务器.md`
2. `01_服务器模型演进/02_IO多路复用_select_poll_epoll.md`

这两篇是主线。前者整理从单进程阻塞到线程池；后者整理 `select / poll / epoll` 与 I/O 线程边界。

### 第二部分：核心机制

1. `02_核心机制/01_TCP基础与socket流程.md`
2. `02_核心机制/02_fd生命周期与连接管理.md`
3. `02_核心机制/03_ThreadPool实现笔记.md`
4. `02_核心机制/04_Selector与Reactor边界.md`

这些不是某个版本独有，而是贯穿所有服务器版本的底层机制。

### 第三部分：工程实践

1. `03_工程实践/01_工程技巧与常用写法.md`
2. `03_工程实践/02_测试压测与调试.md`
3. `03_工程实践/03_代码审阅记录.md`

工程实践用于记录“知道但容易想不到用”的写法、测试方法和阶段性审阅结果。

### 第四部分：阶段复盘与路线

1. `04_阶段复盘与路线/01_复习问题与行动项.md`
2. `04_阶段复盘与路线/02_后续演进路线.md`

用于每次阶段结束后回看，不作为 API 手册。

## 2. 当前学习主线

```text
阻塞 TCP Echo Server
    -> 多进程服务器
    -> 多线程服务器
    -> 线程池服务器
    -> select I/O 多路复用服务器
    -> poll
    -> epoll
    -> Connection / read_buffer / write_buffer
    -> HTTP Server
    -> 带日志、配置、定时器、数据库/API 的后端服务
```

## 3. 维护规则

- 根目录只保留索引和总览，避免堆满零散 md。
- 模型演进类笔记放入 `01_服务器模型演进/`。
- 跨阶段机制放入 `02_核心机制/`。
- 代码技巧、测试、审阅放入 `03_工程实践/`。
- 阶段总结和路线放入 `04_阶段复盘与路线/`。
- 原始材料放入 `原始资料/`，只用于追溯，不作为主要阅读入口。
