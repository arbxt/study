## 2. threadpool 版本审阅

### 正确点

```text
main 负责 accept。
client_fd 使用值捕获提交给线程池。
worker handle 负责 read/write/close。
ThreadPool 锁外执行 task。
write_all 处理短写和 EINTR。
日志输出使用 mutex 防止交叉。
```

### 局限点

```text
每个连接对应一个长期任务。
慢连接会占用 worker。
线程池任务队列可能积压。
不适合作为高并发长期模型。
```

### 学习价值

```text
适合理解线程池、任务队列、条件变量、client_fd 所有权转移。
```

## 3. select_pool 版本审阅

### 正确点

```text
main 只组装 TcpServer、ThreadPool、Selector。
Selector 集中管理 select loop 和 fd 集合。
handle_business 不 read、不 close。
closed_fds_ 延迟删除并 clear。
remove_client 包含 close、FD_CLR、erase、rebuild_max_fd。
select 错误处理考虑 EINTR。
accept 新 fd 时检查 FD_SETSIZE。
```

### 需要修改

```text
main.cpp 顶部注释仍是旧线程池阻塞模型，需要更新。
TcpServer::start 错误路径可以更及时地 close listen_fd_。
Selector 析构可以关闭仍在 client_fds_ 中的 fd。
```

### 当前可接受但要记录风险

```text
worker 直接 write client_fd。
同一 fd 可能被多个 worker 并发 write。
fd 关闭后，旧 worker 任务可能写到复用 fd。
没有连接对象和读写缓冲区。
```

## 4. tcp_bench_short.cpp 审阅

作用：

```text
用于短连接压测。
统计成功/失败/QPS/平均延迟/P95/P99。
```

优点：

```text
可以帮助观察服务器在并发请求下的表现。
比只用 nc 更接近工程测试。
```

限制：

```text
当前是短连接模型。
不能测试长连接、多请求复用、慢客户端、半包等场景。
```

后续可以增加：

```text
长连接压测
慢客户端测试
大消息测试
断开重连测试
```
