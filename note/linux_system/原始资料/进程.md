## 一、遗言函数
1. 遗言函数在进程杀死时被调用，当进程正常终止时，内核会按 “**后注册先执行（LIFO，栈式顺序）** ” 的规则调用所有遗言函数，最后释放进程资源，遗言函数被注册n次就会被调用n次，子进程会继承父进程的遗言函数。
2. `int atexit(void (*function)(void)); #include <stdlib.h>`
3. `int on_exit(void (*function)(int, void *), void *arg); #include <stdlib.h>`
   - 功能：为该进程注册一个遗言函数，
   - 参数：
     function： 遗言函数的入口地址
     arg：传递给遗言函数的参数
   - 返回值：成功返回0，错误返回非0
# 二、进程资源的回收
1. 进程在执行期间占用了一定的资源，在进程终止时需要将资源释放给系统，由父进程负责回收子进程的资源，系统提供了`wait`等函数用于子进程资源的回收。
2. `pid_t wait(int *_Nullable wstatus); #include <sys/wait.h>`
   - 功能：挂起父进程，等待子进程结束，子进程结束后回收子进程资源
   - 参数：返回状态码的存储地址
   - 返回值：成功返回子进程pid，错误返回-1，设置errno
   - `wstatus`存储的是编码后的状态信息，以下为宏及其配套提取宏
     1. `WIFEXITED(wstatus)`：判断子进程是否**正常终止**，正常返回真
        `WEXITSTATUS(wstatus)`：提取正常终止的状态码（0-255）
     
     2. `WIFSIGNALED(wstatus)`：判断子进程是否**被信号终止**
        `WTERMSIG(wstatus)`：提取终止子进程的信号编号；
        `WCOREDUMP(wstatus)`：判断是否产生核心转储文件（`core dump`）
     
     3. `WIFSTOPPED(wstatus)`：判断子进程是否**被暂停**
        `WSTOPSIG(wstatus)`：提取暂停子进程的信号编号
        
     4. `WIFCONTINUED(wstatus)`：判断子进程是否**从暂停恢复**
3. `pid_t waitpid(pid_t pid, int *_Nullable wstatus, int options);`
   - 功能：可等待指定的进程
   - 参数：
     `pid`:等待的指定的pid
     `wstatus`：同上
     `options`：选项参数，可设置非阻塞
   - 返回值：成功返回被终止的子进程的pid，如果WNOHANG被指定，没有子进程终止返回0，错误返回-1
# 三、进程映像更换
- fork复制父进程的PCB，子进程有自己的4G空间，但此时父子进程执行的程序代码一致。进程的映像属于PCB的一个部分，在子进程空间里装入了指定程序的映像，将原来从父进程继承的映像替换，子进程就有了自己的空间，有了自己执行程序的映像。
- `int execve(const char *pathname, char *const _Nullable argv[],char *const _Nullable envp);system call`
  功能：**用新程序完全替换当前进程的代码段、数据段、栈、堆**， 当前进程的 PID 不变，但执行逻辑、资源完全变成新程序
  参数：
    pathname：新程序路径，要求程序为二进制可执行文件或脚本文件
    argv：新程序的命令行参数组，第一个为名字，最后一个为`NULL`
    envp：新程序的环境变量数组，最后一个为`NULL`，`NULL`表示继承当前进程。
  返回值：成功时无返回，错误返回-1，设置errno
- 
```
   #include <unistd.h>

       extern char **environ;

       int execl(const char *pathname, const char *arg, ...
					/*, (char *) NULL */);
       int execlp(const char *file, const char *arg, ...
                       /*, (char *) NULL */);
       int execle(const char *pathname, const char *arg, ...
                       /*, (char *) NULL, char *const envp[] */);
       int execv(const char *pathname, char *const argv[]);
       int execvp(const char *file, char *const argv[]);
       int execvpe(const char *file, char *const argv[], char *const envp[]);
```
- `bash`上执行的为`Linux`，命令分为内部命令和外部命令。外部命令都是独立的可执行程序，内部命令是`bash`的一部分程序，`bash`执行内部命令时不用`fork`。命令可通过`type commend`查看
# 四、环境变量
- 每个进程都有自己的环境变量，环境变量可被子进程继承。可通过`env`命令查看
- 系统提供了一些列函数用于操作当前进程的环境变量`getenv,setenv,unsetenv,putenv,clearenv`
