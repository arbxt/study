# 内存管理
裸板开发：cpu直接访问内存内的数据，使用物理地址而无需操作系统，常见于单片机开发
## 一、虚拟内存管理技术
**虚拟内存（Virtual Memory）** 是由操作系统和硬件（MMU）共同实现的内存抽象技术。 它为每个进程提供独立、连续的虚拟地址空间，而实际映射到的可能是 **物理内存 + 磁盘（Swap）**。
虚拟内存 = “虚拟地址空间” + “地址映射机制” + “页面置换策略”
## 二、为什么要有虚拟内存
直接访问物理内存会导致：
- 进程间内存互相覆盖；
- 内存碎片管理复杂；
- 无法加载大程序；
- 安全性差。
 引入虚拟内存后带来的好处：
- 进程隔离  
- 内存保护  
- 扩展内存空间（比实际RAM更大）  
- 安全性与稳定性提高

## 三、虚拟内存的核心机制

| 层面             | 关键机制          | 作用      |
| -------------- | ------------- | ------- |
| 地址空间抽象         | 每个进程有独立虚拟空间   | 进程隔离    |
| 分页（Paging）     | 虚拟内存划分为固定大小页  | 便于映射与置换 |
| 页表（Page Table） | 记录虚拟页 → 物理页映射 | 地址翻译    |
| TLB            | 缓存页表项         | 加速地址转换  |
| 缺页中断           | 访问未加载页时触发     | 按需加载    |
| 页面置换算法         | 决定换出哪一页       | 提高效率    |
| Swap           | 把冷页放到磁盘       | 扩展虚拟空间  |
## 四、虚拟地址 → 物理地址 转换过程

1. CPU 发出虚拟地址；
2. **MMU（内存管理单元）** 查页表；
3. 若命中：直接转换为物理地址；
4. 若未命中：
   - 触发 **缺页中断（Page Fault）**；
   - 操作系统从磁盘加载页面；
   - 更新页表后重新执行指令。
## 五、分页机制（Paging）

| 名称             | 内容                |
| -------------- | ----------------- |
| 页（Page）        | 虚拟内存的最小单位（一般 4KB） |
| 页框（Frame）      | 物理内存分配单元（大小与页相同）  |
| 页表（Page Table） | 记录虚拟页号与物理页号的映射关系  |

## 七、内存分配与生命周期
- C 程序由函数组成，每个函数在执行时都有自己的栈帧，函数执行完毕时栈帧失效。
- 函数中的**临时变量、局部变量和形参**的空间都分配在**栈帧**里，这些变量的生命周期仅在**函数调用期间**，而进程的生命周期从程序开始到结束。
- **已初始化的静态变量和全局变量**的空间分配在数据段内，未初始化的静态或全局变量的空间分配在 BSS 段内，它们的生命周期与进程相同，程序运行期间一直存在。
- 全局 const 变量储存在只读数据段内，只能读取而不能写入，而局部 const 变量储存在栈内，其生命周期随函数调用周期，可通过指针访问进行读写操作。
- 栈帧的动态创建与销毁，使得局部变量具有临时性，而数据段和 BSS 段的全局和静态变量则常驻内存，提供整个程序运行期间的状态保存。
- 只读数据段用于存储程序中的常量和全局 const 变量，操作系统通过页表设置为只读以防止被修改。
- 堆区用于动态分配的内存，其生命周期由程序员控制，栈区和堆区的读写权限通过内存映射机制管理，保证程序运行时的安全性。
# mmap将物理地址映射到进程虚拟地址
## 一、功能、参数、返回值
- mmap, munmap - map or unmap files or devices into memory，建立/解除文件和虚拟地址间的映射。
- ```
	void *mmap(void addr[.length], size_t length, int prot, int flags,
                  int fd, off_t offset);
  ```

	addr：指定了新的映射地址，虚拟地址；为空时系统指定
	length：指定了映射区域长度
	prot：描述映射区域权限
	flags：标记位
	    MAP_SHRED：对内存内容的更新可以被映射到该区域的其它进程看到，且更新会同步到文件
	    MAP_PRIVATE：私有的，更新对其它进程不可见，也不同步到文件
		MAP_ANONYMOUS：匿名映射，
	fd：可忽略，-1
	offset：可忽略，0
	成功时返回一个指针，指向映射区域首地址。
- ```
	  int munmap(void addr[.length], size_t length);
  ```    
	addr:接触映射的起始地址，必须是页的整数倍
	length:从起始地址开始接触的映射区的长度
	返回值：成功时返回0，其余失败
## 二、文件操作
### 补充
- 可变参数：`int printf(const char *restrict format, ...);` 可变参数的类型和数量取决于前面一个参数
- 可变参数的两种常见约定方式
  1. 固定参数直接指定“数量+类型”
```
     // 示例：sum(count, ...) 中，count是固定参数，指定后续int类型参数的个数
		int sum(int count, ...) {
		    va_list args;
		    va_start(args, count); // 以count为起点，定位可变参数
		    int total = 0;
		    for (int i=0; i<count; i++) {
		        // 必须明确类型为int（通过隐含规则约定）
		        total += va_arg(args, int);
		    }
		    va_end(args);
		    return total;
		}
		// 调用：count=3 明确后续有3个int参数，类型由函数约定为int
		sum(3, 10, 20, 30); 
```
  1. 固定参数是 “格式字符串”，解析 “数量 + 类型”(printf)
```
	// 简化示例：模仿printf的格式解析 
	void my_printf(const char *format, ...) { 
	    va_list args; 
		va_start(args, format); 
	    // 以format为起点 
	    for (int i=0; format[i]!='\0'; i++) { 
		     if (format[i] == '%') { 
			     i++; // 跳过'%'，解析后续占位符 
			     switch(format[i]) { 
				     case 'd': { // 占位符%d → 类型int，数量+1 
					     int num = va_arg(args, int); 
					     printf("int: %d ", num); 
					     break; 
					}
					case 'f': { // 占位符%f → 类型double，数量+1 
						double f = va_arg(args, double); 
						printf("float: %.2f ", f); break; 
					} 
				} 
			} 
		} 
		va_end(args);
	} 
	// 调用：format="%d%f" 解析出2个参数（int+double）
	my_printf("%d%f", 100, 3.14);
```

### open
- `int open(const char *pathname, int flags, .../* mode_t mode */ );`
- 功能：打开一个文件
- 参数：
  pathname：指定了要打开的文件路径名字
  flags：O_RDONLY, O_WRONLY, or O_RDWR，只读，只写，读写三选一；可选创建/状态标记，O_CREAT,O_EXCL,O_TRUNC,O_APPEND等，当模式为creat时需指定mode来设置新文件权限，mode为权限&~umask
- 返回值：成功时返回一个新的文件描述符，错误时返回-1，设置errno
- 文件描述符：文件描述符是操作系统对 I/O 资源的抽象，本质上是进程打开文件表（Open File Table）的数组下标。在bash上启动的进程文件描述符采用0、1 、2分别对应stdin、stdout、stderr
- 进程：程序运行时使用计算机资源的描述，每个进程都有自己的PID，每个进程有自己的控制块PCB，在PCB中记录了进程当前使用资源的状况。文件是计算机的资源，对文件的打开和关闭都会记录到进程的PCB中
- linux文件类型：通过ls -l命令查看文件时输出的第一个字符代表文件类型
  1. `-` 普通文件，用于储存文本、二进制数据、程序代码等，不含文件系统结构信息，仅存储实际数据。
  2. `d` 用于存储文件和子目录的**索引信息**，是 Linux 文件系统的组织核心。本质是特殊文件，包含文件名与对应 inode 的映射关系（类似 “文件夹”）。
  3. `l` 用于指向另一个文件的**快捷方式**，分为硬链接和软链接（符号链接）。**软链接**- 类似 Windows 快捷方式，独立文件，存储目标文件的路径。目标文件删除后，软链接失效。**硬链接**- 与目标文件共享同一个 inode，本质是同一个文件的不同名称。- 与目标文件共享同一个 inode，本质是同一个文件的不同名称。
  4. `p` **管道文件**用于**进程间通信（IPC）** 的临时文件，属于 “先进先出（FIFO）” 队列。数据写入管道后，被读取进程取走即删除，不持久存储。
  5. `c` **字符设备文件**用于访问**按字符流传输数据的设备**（如键盘、鼠标、串口）。数据传输无缓冲，按字节实时处理。
  6. `b` **块设备文件**用于访问**按块（固定大小）传输数据的设备**（如硬盘、U 盘、光盘）。数据传输有缓冲，以块（通常 512B 或 4KB）为单位处理，支持随机访问。
  7. `s` **socket**文件用于**网络通信或本地进程间通信**的特殊文件，支持 TCP/UDP 等协议。通过网络或本地套接字实现进程间数据交换，常用于服务器程序。
- 进程的umask：权限的掩码，通过umask命令查看

### close
- `int close(int fd);` 
- 功能：关闭一个文件描述符
- 参数：fd，关闭的描述符
- 返回值：成功返回0，失败返回-1，设置errno
### read
- `ssize_t read(int fd, void buf[.count], size_t count);`
- 功能：从一个文件描述符读取数据
- 参数：
  fd,要读取的文件描述符
  buf，存储读取到数据的空间的
  count，能读取的数据的最大的数量
- 返回值：返回读取到的字节数，0表示到达文件末尾，错误时返回-1，设置errno
### write
- `ssize_t write(int fd, const void buf[.count], size_t count);`
- 功能：向一个文件描述符中写入数据
- 参数：
  fd,要写入数据的文件描述符
  buf，存储数据内容的地址
  count，请求写入的最大字节数
- 返回值：返回成功写入的字节数，0表示什么都没做，错误时返回-1，设置errno
### lseek
- `off_t lseek(int fd, off_t offset, int whence);`
- 功能：重新定位文件读写的位置
- 参数：
  fd：文件描述符
  offset：偏移量
  whence：偏移起始点，SEEK_SET开头，SEEK_CUR当前，SEEK_END末尾
- 返回值：成功返回距离文件开头的字节数，失败返回-1，设置errno
## 三、文件描述符的复制（输入输出的重定向）
1. `dup`
   - `int(int oldfd) `
   - 功能：复制一个文件描述符
   - 参数：源描述符
   - 返回值：成功返回新的文件描述符，新的文件描述符是最小的未被使用的
2. `dup2`
   - `int(int oldfd , newfd)`
   - 功能：复制一个文件描述符
   - 参数：
     `oldfd`：源描述符
     `newfd`：新的文件描述符
   - 返回值：新的文件描述符
