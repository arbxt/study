# 文件元数据meta
- 文件的元数据就是文件的属性信息
- 文件除了内容还有元数据，每个文件都有且只有一个自己的inode，当两个文件的inode相同时，称这两个文件为硬链接文件；一个文件的内容为另一个文件的inode时，称这个文件为第二个文件的软链接。
- `int stat(const char *path, struct stat *buf),#include <sys/stat.h>`
  - 功能：获取文件meta信息
  - 参数：`path`:文件路径；`buf`：将meta信息存储到buf指定的空间内
  - 返回值：成功0；错误返回-1，设置errno
  - struct stat：存储文件元数据的核心数据结构，包含设备ID，inode，类型和权限等信息
- `struct passwd *getpwuid(uid_t uid),#include <sys/types.h>,#include <pwd.h>`
  - 功能：找到指定uid对应的用户信息
  - 参数：指定了要找的uid
  - 返回值：成功返回对应用户信息，失败返回空，错误时返回空设置errno
# 文件夹操作
- 权限：rwx，读写通过
- 文件夹内容：内部的文件和文件夹
- 文件夹流
- 文件夹操作：`opendir,closedir,readdir...,#include <sys/types.h>,#include <dirent.h>`
  1. `DIR *opendir(const char* name)`
     - 功能：打开一个文件夹，定位在文件夹第一条记录上
     - 参数：name，文件夹路径
     - 返回值：成功返回DIR指针，失败返回空，设置errno
  2. `int closedir(DIR *dirp);`
     - 功能：关闭文件夹
     - 参数：DIR指针，要关闭的文件夹
     - 返回值：成功返回0，失败返回-1，设置errno
 3. `struct dirent *readdir(DIR *dirp)`
     - 功能：读一个文件夹
     - 参数：文件夹DIR指针
     - 返回值：dirent结构体，包含`d_ino,dname[256]`等信息（即文件inode，文件名）
