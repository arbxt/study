## 0. 使用方式

这个目录同时保留两种阅读方式：

```text
00_Linux系统编程总笔记.md
    完整整合版，适合全文搜索、整体复习和阶段回顾。
```

## 3. 当前学习主线

```text
程序构建、链接、调试
    -> 文件与 fd
    -> 进程生命周期
    -> 虚拟内存与地址空间
    -> IPC 与信号
    -> 网络服务器项目中的系统编程实践
```

Linux 系统编程不是单独背 API，而是为了支撑长期主项目：

```text
select + threadpool echo server
    -> poll
    -> epoll
    -> HTTP server
    -> 日志 / 配置 / 定时器 / 数据库 / 缓存
```

## 4. 命名规则

本目录使用“中文为主，必要英文保留”的命名方式：

```text
中文：文件系统、进程控制、虚拟内存、信号、守护进程
保留英文：Linux、fd、IPC、mmap、GDB、ELF、errno、select、threadpool
```

原因：中文名称更适合长期复习和快速定位；专业缩写保留英文可以避免翻译后失真。

## 5. 维护规则

每次补充笔记时，尽量按这个模板追加：

```markdown
## 主题

### 1. 这个知识点属于哪个层次

### 2. 它解决什么问题

### 3. 背后的系统对象

### 4. 常见 API / 命令

### 5. 易错点

### 6. 与服务器项目的关系

### 7. 实践任务
```

不要只记录 API 参数，要记录：

```text
为什么需要它？
它操作的是哪个内核对象？
返回值和 errno 如何处理？
在服务器项目中哪里会遇到？
```
